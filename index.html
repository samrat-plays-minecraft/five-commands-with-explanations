5 Commands with Explanations!

1. Instant bridge using snowballs

/execute @e[type={entityName}] ~ ~ ~ fill ~ ~{number} ~ ~{number} ~{number} ~{number} {blockName}

/execute - Executes a command on behalf of one or more entities.

@e means all entities.

"type" is the entity type (Example= cow, zombie)

entityName is the entity type value.

~ means the current position.

/fill - Fills specific areas (within user-entered coordinates) with specific blocks.

~number means block coordinates added/subtracted in their current position.

blockName is the block name (Example= diamond_block, stone).

Command used:

/execute @e[type=snowball] ~ ~ ~ fill ~ ~-1 ~ ~-1 ~-1 ~-1 diamond_block

2. Converyor belt

/execute @e ~ ~ ~ detect ~ ~{number} ~ {blockName} {dataValue} tp ~ ~ ~{number}

/execute - Executes a command on behalf of one or more entities.

@e means all entities.

~ means the current position.

/detect - Detects blocks at specific coordinates.

~number means block coordinates added/subtracted in their current position.

blockName is the block name (Example= diamond_block, stone).

dataValue means specific blocks of same type (Example= planks: 0 is oak, 1 is spruce, 2 is birch).

/tp means teleport command.

Commands used:

4. Auto-generating parkour

/execute @e ~ ~ ~ detect ~ ~{number} ~ {blockName} {dataValue} title @p actionbar {stringValue}

/execute - Executes a command on behalf of one or more entities.

@e means all entities.

~ means the current position.

/detect - Detects blocks at specific coordinates.

~number means block coordinates added/subtracted in their current position.

blockName is the block name (Example= diamond_block, stone).

dataValue means specific blocks of same type (Example= planks: 0 is oak, 1 is spruce, 2 is birch).

/title - Displays title.

@p means the nearest player.

actionbar means the item name area.

stringValue means the text.

/playsound {soundName} @p ~ ~ ~ {volume}

/playsound - Plays a sound.

soundName is the sound name (Example= random.levelup, mob.zombie.say).

@p means the nearest player.

~ means the current position.

volume is a number value which defines the volume of the sound to be played.

/setblock x y z {blockName} {dataValue}

/setblock - Places a specific block on specific coordinates.

x y z are the coordinate values.

blockName is the block name (Example= diamond_block, stone).

dataValue means specific blocks of same type (Example= planks: 0 is oak, 1 is spruce, 2 is birch).

/fill - Fills specific areas (within user-entered coordinates) with specific blocks.

{start= x y z} are the starting coordinates.

{end= x y z} are the ending coordinates.

blockName is the block name (Example= diamond_block, stone).

dataValue means specific blocks of same type (Example= planks: 0 is oak, 1 is spruce, 2 is birch).
